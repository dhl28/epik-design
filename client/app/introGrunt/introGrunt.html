使用 grunt

概述

  • 为何要用构建工具

    一句话：自动化。对于需要反复重复的任务，例如压缩（minification）、编译、单元测试、linting等，自动化工具可以减轻你的劳动，简化你的工作。当你在 Gruntfile 文件正确配置好了任务，任务运行器就会自动帮你或你的小组完成大部分无聊的工作。

  • 为什么要使用Grunt

    Grunt生态系统非常庞大，并且一直在增长。由于拥有数量庞大的插件可供选择，因此，你可以利用Grunt自动完成任何事，并且花费最少的代价。如果找不到你所需要的插件，那就自己动手创造一个Grunt插件，然后将其发布到npm上吧。

http://www.gruntjs.net/

使用

epik项目使用grunt来构建，沿用了angular-fullstack的默认配置，其中包含

  • injector : 将 `client/{app,components}` 中的所有 js 和 css 文件注入到 `client/index.html` 中。
  • wiredep : 在 `client/index.html` 中注入 bower 组件。
  • useminPrepare: 修改 将 js 和 css 合并压缩到 `dist/public/app` 下的 Grunt 配置，由 `concat`、`uglify`、`cssmin` 组成
  • usemin : 根据 useminPare 配置 和 rev 修改文件中的引用。
  • autoprefixer: 为 `.tmp/` 中的 css 添加前缀。
  • ngtemplates: 将模板文件压缩后添加到 下。
  • concat: 合并
  • cssmin: 压缩 CSS
  • uglify: 压缩混淆 JavaScript
  • rev: 重命名 dist/public 下的资源文件，解决资源版本问题

更详细的代码解释：http://blog.inching.org/2014/11/25/grunt-angular-fullstack-yo/

通过grunt启动项目

  • grunt serve 启动开发环境
  • grunt serve:dist 构建并启动生产环境

grunt.registerTask('serve', function (target) {
  if (target === 'dist') {
    return grunt.task.run(['build', 'env:all', 'env:prod', 'express:prod', 'wait', 'open', 'express-keepalive']);
  }

  if (target === 'debug') {
    return grunt.task.run([
      'clean:server',
      'env:all',
      'concurrent:server',
      'injector',
      'wiredep',
      'autoprefixer',
      'concurrent:debug'
    ]);
  }

  grunt.task.run([
    'clean:server',
    'env:all',
    'concurrent:server',
    'injector',
    'wiredep',
    'autoprefixer',
    'express:dev',
    'wait',
    'open',
    'watch'
  ]);
});
